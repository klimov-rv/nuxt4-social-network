# О проекте Nuxt4 Social Network

## Общая информация

Мини-версия социальной сети на Nuxt 4 с интеграцией Element Plus UI. Проект предназначен для демонстрации работы с авторизацией, профилями пользователей и поиском.

**Стек:**

- Nuxt 4 (с поддержкой TypeScript)
- Element Plus UI
- i18n (интернационализация)
- Vitest (тестирование)
- Nuxt Content (управление контентом)

---

## Структура проекта

### Основные страницы

1. **`/` (index.vue)** - Экран авторизации

   - Кнопки социальной авторизации (Telegram, Google)
   - Поля ввода логина и пароля
   - Переход на страницу профиля
   - **Статус:** ✅ Готово, но авторизация не реальная

2. **`/profile` (profile.vue)** - Профиль текущего пользователя

   - Отображает профиль пользователя "Романа Климова"
   - Загружает данные из content (queryCollection)
   - **Статус:** ✅ Готово

3. **`/search` (search.vue)** - Поиск пользователей

   - Входная поле поиска с иконой
   - Статический список американских штатов (не связан с реальными пользователями)
   - **Статус:** ❌ Не работает - не ищет по пользователям

4. **`/user/[slug]` (user/[slug].vue)** - Профиль другого пользователя
   - Динамическая загрузка профиля по slug'у
   - Отображает уведомление об успешной авторизации
   - **Статус:** ✅ Готово

### Компоненты и структура

```
app/
├── components/
│   ├── shared/       (переиспользуемые компоненты)
│   ├── entities/     (бизнес-логика сущностей)
│   ├── features/     (функциональность)
│   └── widgets/      (виджеты и сложные компоненты)
├── providers/
│   ├── auth/         (провайдер авторизации)
│   ├── i18n/         (интернационализация)
│   └── theme/        (тема приложения)
├── stores/           (Pinia хранилища)
├── pages/            (страницы маршрутизации)
├── layouts/          (макеты страниц)
├── middleware/       (middleware авторизации)
└── composables/      (переиспользуемые логики)
```

## Выявленные проблемы и исправления

### Проблема 1: Поиск не работает

- **Описание:** Search page показывает список американских штатов вместо пользователей
- **Исправление:** Требуется реализовать поиск по пользователям из `data/content/users/`
- **Статус:** 🔄 Будет исправлено на этапе 2

### Проблема 2: Авторизация не реальная

- **Описание:** Кнопки социальной авторизации просто переводят на `/profile` без проверки
- **Статус:** 🔄 Будет реализовано на этапе 3

### Проблема 3: Тесты не настроены

- **Описание:** test/example.spec.ts имеет неправильный синтаксис (mount не может монтировать строки путей)
- **Статус:** 🔄 Будет исправлено на этапе 3

### Проблема 4: Недостаточно пользователей

- **Описание:** Только 2 пользователя для тестирования поиска
- **Статус:** 🔄 Будет расширено до 10 на этапе 2

## Миграция из React/Next.js к Nuxt 4

### Ключевые отличия

- **Маршрутизация:** Файлы в `pages/` автоматически становятся маршрутами
- **Компоненты:** Автоматический импорт из папок в nuxt.config.ts
- **Состояние:** Использование Pinia вместо других решений
- **SSR:** Готово out-of-the-box
- **Content:** Nuxt Content для управления MD файлами
